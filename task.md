# auth4x 基础功能任务清单（logto 预备）

## 目标

为实现 logto 的基础功能准备必要的服务结构、数据模型与基础能力。

## 任务列表

- [x] 明确最小可用功能（MVP）范围：登录、注册、token 颁发、用户信息查询。
- [x] 设计 API 清单与路由（基于 `auth4x.api`）：
  - [x] `POST /api/auth/sign-in`
  - [x] `POST /api/auth/sign-up`
  - [x] `POST /api/auth/token`
  - [x] `GET /api/users/me`
- [x] 定义数据模型与存储方案：用户、凭证、会话、刷新 token。
- [x] 选择配置结构与环境变量规范（如数据库连接、JWT 配置）。
- [x] 引入基础中间件：日志、鉴权、错误处理、请求追踪。
- [x] 准备初始数据库迁移与种子数据脚本。
- [ ] 编写基础测试策略：单元测试与接口测试。

## 里程碑

1. 完成 API 设计与数据模型定义。
2. 跑通注册/登录/token 颁发闭环。
3. 补齐用户信息与基础鉴权能力。
4. 完成基础测试策略与样例覆盖。
